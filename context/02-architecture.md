## Tech stack

| Layer | Technology | Role |
|-------|------------|------|
| Framework | Astro 6 | Content collections, build, routing |
| Runtime | Bun | Local dev + build (Node 20+ on CF Pages) |
| Language | TypeScript | All engine code |
| Markdown | remark + rehype + remark-directive | Content pipeline |
| Typesetting | pretext (chenglou/pretext) | Per-line measurement + breaking |
| Hyphenation | Hyphenopoly (en-gb) | Soft-hyphen injection (Node-side) |

| Images | Sharp + thumbhash | AVIF/WebP/fallback variants + blur placeholders |

| Domain | pilcrow.page | Canonical (CNAME → pilcrow-eiq.pages.dev) |

## Build pipeline (the order is sacred)

```
1. bun run build  →  astro build
2. Astro content sync       →  rehype-images.ts runs Sharp; writes public/_images/<slug>-Nw.<fmt>
3. Astro generates HTML     →  remark/rehype plugins fire in this order:
                                a. remarkDirective + remarkPullquote + remarkSidenote
                                b. (remark-gfm@4.0.1, default in Astro, gfm: true)
                                c. rehypeFootnoteMark + rehypeHoistSidenotes + rehypeImages
4. Astro writes dist/posts/**/index.html
5. astro:build:done hook    →  pilcrow-typeset.ts integration runs:
                                a. spawns Playwright Chromium
                                b. for each .post-body, replace <p> with per-line <span class="pt-line">
                                c. preserves sidenote <sup> markers via buildLineSpansHTML helper
                                d. orphan guard re-runs paragraphs with offending SHYs stripped
6. Astro copies public/     →  dist/ (including _images/)
7. dist/ ready              →  CF Pages deploys
```

**Bypass:** `PILCROW_SKIP_TYPESET=1 bun run build` skips step 5 (useful for content iteration or CI without Chromium). OG card generation is a separate static route and still runs.

## System boundaries (do not cross)

- **Markdown plugins** own structural HTML transformation. They never measure typography.
- **Playwright pass** owns measurement and per-line wrapping. It never restructures HTML beyond `<p>` → `<span class="pt-line">…</span>` replacement.
- **`public/styles/global.css`** is the single source of truth for measurement-critical rules. `playwright.ts` reads it via `loaderHTML` at build time. Two copies of these rules anywhere = drift = silent miscalculation.
- **`packages/create-pilcrow/template/`** is a snapshot of the engine at a point in time. Engine and template can drift; sync is currently manual (tracked in `NOTES.md` as v1.x candidate).

## Storage model

- **Posts** in `src/content/posts/*.md` (Astro content collection)
- **Images** authored as `./images/photo.jpg` relative to the post; Sharp processes during content sync to `public/_images/`
- **Variants** at `dist/_images/<slug>-<width>w.<format>` (always copied via public/)
- **OG cards** generated as static routes (`/og/[slug].png` and `/og/index.png`)
- **Astro content cache** at `node_modules/.astro/data-store.json` — devalue-serialized rendered HTML by source digest

## Tool interaction rules

- **Hyphenopoly ⇄ pretext:** Hyphenopoly inserts SHYs Node-side; pretext consumes them in the browser. Pretext is grapheme-aware, not syllable-aware → orphan guard wrapper required (see `03-code-standards.md` for the full policy and `NOTES.md` for upstream issue #162).
- **Sharp ⇄ Astro dist:** **NEVER write Sharp output directly to `dist/`.** Astro cleans `dist/` at the start of each build. Always write to `public/_images/`; Astro copies to `dist/` automatically.
- **Satori ⇄ Fraunces:** Satori's opentype.js parser cannot handle multi-axis fvar tables. Use the static instance `src/assets/fonts/Fraunces144pt-Bold.ttf`, NOT the variable file.
- **OG endpoints ⇄ Astro prerender:** font path must be `process.cwd() + 'src/assets/fonts/...'`. `import.meta.url`-relative paths resolve to `dist/.prerender/chunks/` during prerender and break.
- **Playwright ⇄ measurement CSS:** `loaderHTML` in `playwright.ts` reads from `public/styles/global.css`. If a measurement-critical rule is added, extend `readMeasurementCSS()` to extract it.
- **Astro content cache ⇄ debugging plugins:** when a rehype plugin appears not to execute, **clear `node_modules/.astro/data-store.json`** (not just `.astro/` or `.vite/`). It's almost always a cache hit, not a pipeline bug.

## Invariants

1. Reader receives zero JS except the optional 416-byte image blur-up (and only when `hasImages: true` is set in front-matter).
2. The `--prose-measure: 65ch` CSS custom property is the single source of truth for prose column width. Hard error if absent in `readMeasurementCSS()`.
3. `--paper #fafaf7`, `--ink #1a1a1a`, `--muted #6c6a63`, `--rule #d4d0c4`, `--accent #b13a2e` are the only colour tokens. No hex literals in component CSS.
4. Author names inside `<cite>` and digit-only `<sup>`/`<sub>` content are never hyphenated. (Hyphenopoly skip list maintained in `src/lib/typeset/hyphenate.ts`.)
5. Markdown plugin order in `astro.config.mjs` is `remarkDirective → remarkPullquote → remarkSidenote` (remark) and `rehypeFootnoteMark → rehypeHoistSidenotes → rehypeImages` (rehype). Reordering breaks the sidenote hoist.
6. `pretextReady` in `playwright.ts` is gated on `document.fonts.ready` so Fraunces metrics are available before any pretext canvas measurement.
7. `splicePostBody` and the body-extraction logic in `pilcrow-typeset.ts` use depth-counting div-balanced scanners, NOT regex — primitives that introduce nested `<div>` (e.g. `.footnotes-mark`) break regex.
8. CSS `::before { content: 'U+glyph' }` is forbidden for decorative Unicode (read aloud by NVDA/JAWS/VoiceOver). Use a real DOM element with `aria-hidden="true"` instead.

## Environment & deploy

- **Local:** `bun install` → `bun run dev` (no typeset) or `bun run build` (full pipeline)
- **Cloudflare Pages build command:** `npm install -g bun && bun install && bun run build`
- **Output:** `dist/`
- **Node version:** 20+
- **Playwright Chromium:** installed via `postinstall` hook (`playwright install chromium`); ~170 MB download on first build
- **Custom domain:** pilcrow.page (CNAME @ → pilcrow-eiq.pages.dev, free SSL)
- **Emergency build:** `PILCROW_SKIP_TYPESET=1` for non-typeset deploy if Chromium fails
