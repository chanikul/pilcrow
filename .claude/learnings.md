# Pilcrow v1 ‚Äî Agent Learnings

Append-only. Format: `- [YYYY-MM-DD] [agent or area]: [the lesson, one sentence]`

- [2026-04-29] typography-architect: Pretext's `continueSoftHyphenBreakableSegment` packs graphemes after a soft-hyphen, so the materialized line can end with `-<suffix>` (e.g. `ital-i`) not just `-`; orphan detection must handle both cases, not only lines that end with a bare `-`.
- [2026-04-29] typography-architect: Stripping SHYs in document order (first SHY first) causes collateral grapheme-break regressions on unrelated words; always use targeted stem-search (`stem + U+00AD`) to find the specific SHY to strip.
- [2026-04-29] typography-architect: The orphan guard helpers (`guardFlat`, `guardRich`) must be defined inside `page.evaluate()`'s callback ‚Äî they run in the browser context and cannot be imported as Node modules; keep them co-located in `playwright.ts`.
- [2026-04-29] typography-architect: When adding `<cite>` (pull quote attribution) to the HTML, Hyphenopoly will hyphenate author names inside `<cite>` unless `<cite>` is explicitly added to the skip list in `hyphenate.ts` alongside `<code>` ‚Äî author names must never be hyphenated.
- [2026-04-30] design-critic: Screenshot pattern-matching for hyphenation orphans is unreliable ‚Äî a hyphenated suffix at line-end (e.g. `prim-`, `destina-`, `genu-`, `walkRichIn-`) reads visually as a standalone fragment but is actually the tail of a 100%+ fill line; char-count `.pt-line` widths is the authoritative method (two incidents: `walkRichIn-` on `inline-markup.md` ¬∂3, and five false-positive orphans across all posts ‚Äî all resolved as full-fill lines under char-count inspection); design-critic.md Critical rule 7 now mandates char-count first.
- [2026-04-29] typography-architect: The `splicePostBody` regex in `pilcrow-typeset.ts` used a non-greedy `[\s\S]*?` match that stops at the FIRST `</div>` inside `.post-body`; any primitive that introduces a nested `<div>` inside the post body (e.g. the `.footnotes-mark` pilcrow glyph element) will break it ‚Äî replace with a depth-counting balanced-div scanner, not a regex.
- [2026-04-29] typography-architect: CSS `::before { content: 'U+glyph' }` with Unicode characters (¬∂, *, etc.) is read aloud by NVDA ("pilcrow"), JAWS, and VoiceOver when verbose; always use a real DOM element with `aria-hidden="true"` for decorative Unicode glyphs instead of generated content.
- [2026-04-29] typography-architect: Astro includes `remark-gfm@4.0.1` by default with `gfm: true` ‚Äî GFM footnotes work without adding it to `astro.config.mjs`; do not add it as an explicit plugin as it will run twice.
- [2026-04-30] typography-architect: CSS Grid grid-column can only target direct children of the grid container ‚Äî an `<aside>` nested inside `<span>` inside `<p>` cannot use grid-column directly; use float + negative margin-right (`calc(-width - gap)`) instead, which is the classical magazine-float technique and works with no JS.
- [2026-04-30] typography-architect: When `.post-body` gains a Grid wider than the `.post` wrapper's `max-width: 65ch`, the right-margin column overflows or is clipped; the fix is to expand `.post` to `max-width: calc(65ch + gap + sidenote-width)` at the relevant breakpoint in a `@media (min-width: breakpoint)` rule ‚Äî not by removing the prose max-width.
- [2026-04-30] typography-architect: The `readMeasurementCSS()` fallback for `postBodyMaxWidth` is `'65ch'` ‚Äî if `.post-body` loses its `max-width` property (e.g. after Grid restructuring), the fallback still correctly constrains body paragraphs in the loader page to 65ch; no change to the extraction logic is needed as long as the fallback matches the prose column width.
- [2026-04-30] typography-architect: Astro 6 content collections (glob loader) use a persistent data store cache at `node_modules/.astro/data-store.json` (devalue-serialized) ‚Äî when the digest of a source file hasn't changed, Astro returns the cached rendered HTML and NEVER calls the markdown processor; a rehype plugin that appears not to execute is almost always a cache hit, not a pipeline bug; clear `node_modules/.astro/data-store.json` (not just `.astro/` or `node_modules/.vite`) to force a fresh render.
- [2026-04-30] typography-architect: Errors inside rehype plugins running in Astro's content-layer glob loader are caught silently by the loader's try/catch and logged only as `[glob-loader] Error rendering X.md: ...`; the build succeeds but `entry.rendered` is set to `undefined`, producing empty post bodies ‚Äî always run a full clean build with `node_modules/.astro/data-store.json` deleted when debugging plugin execution.
- [2026-05-01] typography-architect: npm strips files named `.gitignore` when packing the package ‚Äî when bundling a template directory inside an npm package, store the template's gitignore as `gitignore` (no leading dot) and have the CLI rename it to `.gitignore` when copying into the user's target directory; affects any `create-*` npm package that needs to ship a `.gitignore` in its template.
- [2026-05-01] typography-architect: `packages/` directory inside an Astro project is automatically ignored by Astro's build pipeline (Astro only processes `src/`, `public/`, and configured directories) ‚Äî no monorepo workspace config needed when keeping a sibling create-package alongside the main Astro project.
- [2026-05-01] typesetter: Sidenote markers (`<sup class="sidenote-marker">`) appended AFTER `display:block` `.pt-line` spans inside `<p>` trigger CSS anonymous block wrapping, rendering the marker as a standalone element on its own visual line ("stray 1 in accent colour"); fix is to inject the marker HTML inside the last `.pt-line` span via `buildLineSpansHTML(inners, markers)` helper in `playwright.ts`.
- [2026-05-06] typesetter: Astro 6 + `moduleResolution: 'Bundler'` does NOT auto-pick up `@types/node` from a workspace-hoisted devDep alone ‚Äî explicit `"types": ["node"]` in the package's `tsconfig.json` was required alongside the devDep declaration to resolve `node:fs/promises`, `process`, etc. (Discovered during `pilcrow-typeset` extraction.)
- [2026-05-06] typesetter: `unist-util-visit` v5 with hast trees passes `Root | Element | undefined` as the `parent` argument to the visitor callback, NOT `Parent | null`. The codebase had typed it as `Parent | null` since the original write ‚Äî the typed package extraction surfaced the mismatch. Pattern for any future `visit()` callback against a hast tree: import `Root` and `Element` from `'hast'`, type parent as `Root | Element | undefined`, and guard with `parent === undefined`.
- [2026-05-06] typesetter: Sprint extraction sub-tasks can land their work eagerly during file moves rather than as discrete steps ‚Äî always read the current state of the target file before planning a write, the spec may already be satisfied. Cosmetic churn against an API-locked surface is worse than no edit. (Verified during `pilcrow-typeset@0.1.0` sub-task 3: `typeset()` and four re-exports were already in place from sub-tasks 1‚Äì2; correct response was to verify line-by-line and confirm gates, not rewrite.)
- [2026-05-06] typesetter: When verifying that a build-time text transformation actually ran (Hyphenopoly soft-hyphen injection, or any pass that emits markers a downstream consumer may eat), grep the entire `dist/` corpus, not a single representative file. A single-post grep can return zero matches even when the transformation worked correctly, because every emitted marker can land at a break position where the downstream pass (in this case pretext) consumes it back out. Corpus-wide grep is the only reliable signal. (Discovered during sub-task 4 verification: `dist/posts/the-cheapest-signal/index.html` had 0 U+00AD characters and almost triggered a false-positive "stop and report"; all-posts grep surfaced 20 surviving SHYs across the corpus, plus mid-syllable line breaks in every post ‚Äî the actual proof.)
- [2026-05-06] typesetter: Sprint-shaped work (multi-deliverable, time-boxed, sub-tasks under ~30 minutes each) does not need per-sub-task feature specs. The sprint plan itself functions as the spec. Reserve `context/feature-specs/NN-<name>.md` for unsprinted standalone features where the planning artefact and the spec artefact would otherwise be the same document written twice.
