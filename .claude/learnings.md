# Pilcrow v1 — Agent Learnings

Append-only. Format: `- [YYYY-MM-DD] [agent or area]: [the lesson, one sentence]`

- [2026-04-29] typography-architect: Pretext's `continueSoftHyphenBreakableSegment` packs graphemes after a soft-hyphen, so the materialized line can end with `-<suffix>` (e.g. `ital-i`) not just `-`; orphan detection must handle both cases, not only lines that end with a bare `-`.
- [2026-04-29] typography-architect: Stripping SHYs in document order (first SHY first) causes collateral grapheme-break regressions on unrelated words; always use targeted stem-search (`stem + U+00AD`) to find the specific SHY to strip.
- [2026-04-29] typography-architect: The orphan guard helpers (`guardFlat`, `guardRich`) must be defined inside `page.evaluate()`'s callback — they run in the browser context and cannot be imported as Node modules; keep them co-located in `playwright.ts`.
- [2026-04-29] typography-architect: When adding `<cite>` (pull quote attribution) to the HTML, Hyphenopoly will hyphenate author names inside `<cite>` unless `<cite>` is explicitly added to the skip list in `hyphenate.ts` alongside `<code>` — author names must never be hyphenated.
- [2026-04-30] design-critic: Screenshot pattern-matching for hyphenation orphans is unreliable — a hyphenated suffix at line-end (e.g. `prim-`, `destina-`, `genu-`, `walkRichIn-`) reads visually as a standalone fragment but is actually the tail of a 100%+ fill line; char-count `.pt-line` widths is the authoritative method (two incidents: `walkRichIn-` on `inline-markup.md` ¶3, and five false-positive orphans across all posts — all resolved as full-fill lines under char-count inspection); design-critic.md Critical rule 7 now mandates char-count first.
- [2026-04-29] typography-architect: The `splicePostBody` regex in `pilcrow-typeset.ts` used a non-greedy `[\s\S]*?` match that stops at the FIRST `</div>` inside `.post-body`; any primitive that introduces a nested `<div>` inside the post body (e.g. the `.footnotes-mark` pilcrow glyph element) will break it — replace with a depth-counting balanced-div scanner, not a regex.
- [2026-04-29] typography-architect: CSS `::before { content: 'U+glyph' }` with Unicode characters (¶, *, etc.) is read aloud by NVDA ("pilcrow"), JAWS, and VoiceOver when verbose; always use a real DOM element with `aria-hidden="true"` for decorative Unicode glyphs instead of generated content.
- [2026-04-29] typography-architect: Astro includes `remark-gfm@4.0.1` by default with `gfm: true` — GFM footnotes work without adding it to `astro.config.mjs`; do not add it as an explicit plugin as it will run twice.
- [2026-04-30] typography-architect: CSS Grid grid-column can only target direct children of the grid container — an `<aside>` nested inside `<span>` inside `<p>` cannot use grid-column directly; use float + negative margin-right (`calc(-width - gap)`) instead, which is the classical magazine-float technique and works with no JS.
- [2026-04-30] typography-architect: When `.post-body` gains a Grid wider than the `.post` wrapper's `max-width: 65ch`, the right-margin column overflows or is clipped; the fix is to expand `.post` to `max-width: calc(65ch + gap + sidenote-width)` at the relevant breakpoint in a `@media (min-width: breakpoint)` rule — not by removing the prose max-width.
- [2026-04-30] typography-architect: The `readMeasurementCSS()` fallback for `postBodyMaxWidth` is `'65ch'` — if `.post-body` loses its `max-width` property (e.g. after Grid restructuring), the fallback still correctly constrains body paragraphs in the loader page to 65ch; no change to the extraction logic is needed as long as the fallback matches the prose column width.
- [2026-04-30] typography-architect: Astro 6 content collections (glob loader) use a persistent data store cache at `node_modules/.astro/data-store.json` (devalue-serialized) — when the digest of a source file hasn't changed, Astro returns the cached rendered HTML and NEVER calls the markdown processor; a rehype plugin that appears not to execute is almost always a cache hit, not a pipeline bug; clear `node_modules/.astro/data-store.json` (not just `.astro/` or `node_modules/.vite`) to force a fresh render.
- [2026-04-30] typography-architect: Errors inside rehype plugins running in Astro's content-layer glob loader are caught silently by the loader's try/catch and logged only as `[glob-loader] Error rendering X.md: ...`; the build succeeds but `entry.rendered` is set to `undefined`, producing empty post bodies — always run a full clean build with `node_modules/.astro/data-store.json` deleted when debugging plugin execution.
- [2026-05-01] typography-architect: npm strips files named `.gitignore` when packing the package — when bundling a template directory inside an npm package, store the template's gitignore as `gitignore` (no leading dot) and have the CLI rename it to `.gitignore` when copying into the user's target directory; affects any `create-*` npm package that needs to ship a `.gitignore` in its template.
- [2026-05-01] typography-architect: `packages/` directory inside an Astro project is automatically ignored by Astro's build pipeline (Astro only processes `src/`, `public/`, and configured directories) — no monorepo workspace config needed when keeping a sibling create-package alongside the main Astro project.
