# pilcrow v1 — agent instructions

This is the v1 Astro-based scaffold. The master plan above is canonical for all architecture and design decisions.

v1 typesetting runs at build-time only via `src/integrations/pilcrow-typeset.ts`. Set `PILCROW_SKIP_TYPESET=1` to bypass (useful during content iteration or CI without Chromium). The integration hooks `astro:build:done`, walks `dist/posts/**/index.html`, and replaces each `<p>` inside `.post-body` with per-line `<span class="pt-line">` elements using pretext. Dev mode (`bun run dev`) is unaffected — typesetting only runs during `bun run build`.

Astro 6 note: Astro 6 removed the legacy `src/content/config.ts` location. The collection config lives at `src/content.config.ts` and requires a `loader` (using `glob` from `astro/loaders`). Rendering uses `render(post)` imported from `astro:content`, not `post.render()`. Collection entries use `post.id` (not `post.slug`) as the URL slug param.

Rich-inline: `src/lib/typeset/playwright.ts` uses pretext's `prepareRichInline` / `walkRichInlineLineRanges` / `materializeRichInlineLineRange` to preserve inline markup (`em`, `strong`, `a`, `code`, `sub`, `sup`) across line breaks. Any paragraph containing an element outside that whitelist triggers a per-paragraph flat-pretext fallback with a loud `[pilcrow]` build-time warning naming the post path and offending tag.

Drop cap: the lede paragraph (first non-empty `<p>` in `.post-body`) gets a `<span class="drop-cap">` by default. Opt out per-post with `dropCap: false` in front-matter. The integration reads a `<meta name="pilcrow:drop-cap" content="true|false">` tag emitted by `Post.astro` / `Base.astro`. Float-aware line widths are computed via `layoutNextLineRange` (flat path) and `layoutNextRichInlineLineRange` (rich-inline path). Changing the cap CSS (font-size, padding, margin) requires `bun run build` to re-measure the float box.

Editorial theme (shipped 2026-04-29): Fraunces body + h1 (700, opsz 144, letter-spacing -0.015em) and Inter on `<time>` only; five `:root` palette tokens (`--paper: #fafaf7; --ink: #1a1a1a; --muted: #6c6a63; --rule: #d4d0c4; --accent: #b13a2e;`); measure 65ch; `loaderHTML` in `playwright.ts` reads measurement-critical rules from `public/styles/global.css` at build time (single source of truth); `__pretextReady` is gated on `document.fonts.ready` so Fraunces metrics are available before any pretext canvas measurement.

Hyphenation runs Node-side via Hyphenopoly (en-gb) before pretext; soft hyphens injected per the inline-element policy in `src/lib/typeset/hyphenate.ts`. Config: leftmin 3, rightmin 3, minWordLength 6. `<code>` content is skipped; `<sub>`/`<sup>` only hyphenated if `textContent.length > 1`.

Orphan guard (added 2026-04-29): `src/lib/typeset/playwright.ts` contains `guardFlat` and `guardRich` helper functions (inside `page.evaluate()`) that detect short right fragments at hyphen breaks and re-run pretext with the offending soft hyphen stripped. Threshold: 4 chars. Detects both clean breaks (`fail-|ure`) and packed-grapheme breaks (`ital-i|cs`). Falls back to no-hyphenation for a paragraph if no SHY strip resolves the orphan. This is a Pilcrow-local mitigation pending upstream `softHyphenMode: 'strict'` in pretext — see NOTES.md for the upstream issue draft.

Pull quote directive (added 2026-04-29): `:::pullquote ... :::` (single token, no label). Attribution via a paragraph inside the block starting with `— Name` (em-dash + space); the plugin strips the prefix and moves the content into `<footer><cite>Name</cite></footer>`. Single paragraph body for v1; extra body paragraphs emit a build-time warning and are dropped. Plugin: `src/plugins/remark-pullquote.ts`. Registered in `astro.config.mjs` markdown.remarkPlugins: `[remarkDirective, remarkPullquote]` (order matters). All CSS in `public/styles/global.css` `.pullquote` selectors — design: 1.5× body / Fraunces italic 400 / -0.5em text-indent (hanging quote) / 1.4 line-height / 50ch / 2.5rem auto margin / Fraunces small-caps for cite / 0.85× / `var(--muted)` / no chrome. `cite::before { content: "— "; }` auto-inserts the em-dash so source text doesn't contain it manually. `<cite>` added to hyphenate.ts skip list. Drop cap gate: `!p.closest('aside')` prevents pull quote paragraphs from consuming the `isLede` flag.

Footnotes (added 2026-04-29): GFM `[^N]` / `[^label]:` syntax — enabled by Astro's default `remark-gfm@4.0.1` (`gfm: true`), no new dependencies. HTML output is GFM canonical: `<sup><a data-footnote-ref>N</a></sup>` for inline markers; `<section data-footnotes class="footnotes"><ol><li id="user-content-fn-N"><p>…↩</p></li></ol></section>` for the list. Sub/sup hyphenation rule (D7): `src/lib/typeset/hyphenate.ts` changed from `textContent.length > 1` to `!/^\d+$/.test(innerText)` — digits-only content is never hyphenated regardless of length (footnote markers "1", "12", etc. stay clean). Drop cap gate extended: `!p.closest('.footnotes')` added alongside `!p.closest('aside')` in both the lede branch and the `isLede`-consumption else branch of `playwright.ts`. Marker colour inherits from `a { color: var(--accent) }` cascade — no new colour rule. Section break: a `<div class="footnotes-mark" aria-hidden="true">¶</div>` element injected by `src/plugins/rehype-footnote-mark.ts` (registered in `astro.config.mjs` `markdown.rehypePlugins`); CSS pseudo-element was rejected because Unicode glyph content in `::before` is read aloud by major screen readers. `readMeasurementCSS()` in `playwright.ts` extended for `.footnotes p` font-size (0.875em) and line-height (1.5) — single source of truth. `splicePostBody` and the body-extraction logic in `pilcrow-typeset.ts` upgraded from non-greedy regex to depth-counting div-balanced scanners (required because `.footnotes-mark` introduces a nested `<div>` inside `.post-body`). Test post: `src/content/posts/footnotes.md`.

RSS + sitemap (added 2026-04-30): `@astrojs/rss` and `@astrojs/sitemap` are regular dependencies (not devDeps — Astro's convention for integrations). `site: 'https://pilcrow.press'` is set in `astro.config.mjs` `defineConfig()` — sitemap and the RSS `context.site` both read from this. The sitemap integration is registered in `integrations: [mdx(), sitemap(), pilcrowTypeset()]`. The RSS feed lives at `src/pages/rss.xml.ts`; it uses the same draft filter as `index.astro` (`import.meta.env.PROD ? !data.draft : true`) and emits title + description + pubDate + link per item (no summary excerpt, no full HTML — D1=A). Channel metadata: title "Pilcrow", description "Editorial typesetting for the open web." Content depth is deliberately minimal: Pilcrow's typeset page is the artifact, RSS is the envelope. To add a post to the feed: omit `draft: true` from front-matter, or set `draft: false`. `@astrojs/sitemap` crawls all generated routes — it does not apply the draft filter; all built post pages appear in `sitemap-0.xml` regardless of draft status, because Astro generates their HTML regardless.

OG image generation (added 2026-04-30): Per-post and index social-share cards via Satori + @resvg/resvg-js (both devDependencies — build-time only, zero reader JS). Card dimensions: 1200×630 (D2=A). Layout: D3=A title-dominant minimalist — large Fraunces title left-set, muted ¶ glyph bottom-right corner at 0.45× title size (midpoint of D user's 0.4–0.5× range). Palette: PAPER=#fafaf7, INK=#1a1a1a, MUTED=#6c6a63 (glyph colour, not ACCENT — D5=A identity-not-emphasis). No pubdate on cards (bias toward restraint; date in corner would split the muted hierarchy between two elements). No description text on post cards (D3=A click-through to the typeset page). Index card uses the RSS channel description as its title text (D6=Yes). Template: `src/lib/og/card.ts` — exports `renderCard(title: string): Promise<Buffer>`. Font: static TTF instance `src/assets/fonts/Fraunces144pt-Bold.ttf` (Fraunces 144pt Bold, master branch); the variable TTF (4-axis fvar) was tried but Satori's opentype.js parser cannot handle multi-axis fvar tables — use the static 144pt-Bold instance, not the variable file. Font path strategy: `process.cwd() + 'src/assets/fonts/...'` (project root) — NOT `import.meta.url`-relative (that resolves to dist/.prerender/chunks/ during Astro's prerender step). Endpoints: `src/pages/og/[slug].png.ts` (per-post, getStaticPaths mirrors draft filter from index.astro and rss.xml.ts), `src/pages/og/index.png.ts` (single, channel-description text). `Base.astro` emits `og:image`, `og:image:width` (1200), `og:image:height` (630), `twitter:card` (summary_large_image), `twitter:image` — post pages via slug prop through Post.astro → Base.astro; index page via no-slug fallback to /og/index.png. Adding a post to OG generation: same as RSS/sitemap/index — omit `draft: true`. `PILCROW_SKIP_TYPESET=1` bypasses typesetting but OG card generation still runs (it's a separate static route, not the pilcrow-typeset integration).

Sidenote directive (added 2026-04-30, corrective rebuild completed 2026-04-30): `:::sidenote ... :::` container directive. Two-plugin pipeline: `src/plugins/remark-sidenote.ts` (remark) emits `<span class="sidenote-ref"><sup class="sidenote-marker"></sup><aside class="sidenote"><p>…</p></aside></span>` at the block level; `src/plugins/rehype-hoist-sidenotes.ts` (rehype, registered after `rehypeFootnoteMark` in `markdown.rehypePlugins`) hoists the `<aside>` to become a direct child of `.post-body` and moves the `<sup>` into the anchor `<p>`, emitting: `<p>…prose…<sup class="sidenote-marker" data-sidenote-id="N"></sup></p><aside class="sidenote" data-sidenote-id="N"><p>…</p></aside>`. The `<span class="sidenote-ref">` wrapper is removed entirely. The hoist is required because CSS `grid-column` only targets direct children of the Grid container — a nested `<aside>` cannot use `grid-column: 4`. Layout: D3=A — 4-column CSS Grid (`0 65ch 2rem 25ch`) on `.post-body`; `--prose-measure: 65ch` custom property is the single source of truth for prose column width (hard error if absent in `readMeasurementCSS()`). The `.post` wrapper expands to `calc(65ch + 2rem + 25ch)` on `@media (min-width: 1100px)`. Desktop: `grid-column: 4` on `aside.sidenote`. Mobile (≤1099px): `display: block` fallback. Drop cap gate: `!p.closest('aside.pullquote')` + `!p.closest('.footnotes')` + `!p.closest('aside.sidenote')`. Marker preservation: `playwright.ts` saves `<sup class="sidenote-marker">` outerHTML before each `p.innerHTML =` assignment and re-appends after (all 4 paths). Astro 6 cache note: Astro 6 content-layer caches rendered HTML in `node_modules/.astro/data-store.json` (devalue format) — when the digest matches, the markdown processor is skipped entirely. Clear this file (not just `.astro/` or `node_modules/.vite`) to force re-render when debugging plugin changes. Test post: `src/content/posts/sidenotes.md` (7 variants).

Image pipeline (added 2026-04-30): `![alt](./images/photo.jpg)` Markdown syntax → `<figure class="pilcrow-figure"><picture><source avif><source webp><img></picture>[<figcaption>]</figure>`. Pipeline: `src/plugins/rehype-images.ts` (rehype plugin, registered in `astro.config.mjs` markdown.rehypePlugins after `rehypeHoistSidenotes`) calls `src/lib/images/process.ts` which uses Sharp (transitive dep, already present as `sharp@0.34.5`) to generate AVIF + WebP + original-format fallback variants at 640/1280/1920px, and generates a thumbhash PNG data URL at build time via the `thumbhash` npm package (explicit dep). Variants written to `public/_images/` during Astro's content-sync phase; Astro copies `public/` → `dist/` during build, so variants are always present at `dist/_images/`. Breaking: never write Sharp output to `dist/` directly — Astro cleans `dist/` at the start of each build, destroying files written during content sync. Output path: `public/_images/<slug>-<width>w.<format>` where slug is the source filename lowercased with non-alphanumeric chars replaced by hyphens. Sharp reports AVIF source files as format `'heif'` (not `'avif'`) — the `fallbackFormat` function in `process.ts` maps `'heif' → 'avif'`. Thumbhash: decoded to a PNG data URL at build time; stored as `data-placeholder` on `<figure>` so the browser blur-up script needs no decoder (keeps the script under 1KB). Alt-text policy: warn-not-fail — emits `[pilcrow] WARNING: image without alt — <slug> — <filename>` to stderr, sets `alt=""` (WCAG correct for decorative), omits figcaption. Aspect ratio: D3=A original-only, set as inline `style="aspect-ratio: W / H"` on `<figure>` to prevent CLS. Blur-up script: inline `<script>` in `Post.astro`, emitted only when `hasImages: true` is set in the post's front-matter — reads `data-placeholder` from each `figure.pilcrow-figure`, sets it as `background-image` on the figure, fades in the real `<img>` on load via `.loaded` CSS class, then removes the placeholder background. Script size: 416 bytes (budget: 1KB). Script is a no-op if JS is disabled (loading="lazy" still works). `hasImages` field added to content schema in `src/content.config.ts`. Test post: `src/content/posts/images.md` (draft: true). Add `public/_images/` to `.gitignore` — these are build artifacts.

Deploy via Cloudflare Pages git-integration (preferred — auto-builds on push) or CLI (`bunx wrangler pages deploy dist --project-name pilcrow-press`). Pilcrow footer link (added 2026-05-01): every page shows `<footer class="pilcrow-footer"><a href="https://pilcrow.press" rel="noopener">Typeset with Pilcrow ¶</a></footer>` rendered in `Base.astro` (conditionally on `siteConfig.showPilcrowFooter`). Site-level config lives in `src/config/site.ts` — export `siteConfig.showPilcrowFooter: boolean`. Default `true` (growth loop per master plan §12). Set to `false` to hide the footer entirely. CSS in `public/styles/global.css` `.pilcrow-footer` selectors — centred colophon register, Inter 0.85em `var(--muted)`, max-width 65ch, 4rem top margin, 2rem bottom margin. Link colour overrides accent to `var(--muted)` (understated; brand mark is the text, not the colour). No `target="_blank"` (same-tab navigation, editorial register).
