# Hello, Pilcrow

This is a fixture post that exercises the build-time typesetting plugin in a Next.js App Router project. Each paragraph in this document should arrive at the rendered HTML wrapped in per-line spans with the class `pt-line`, computed from the column width and font shorthand declared by the page's CSS.

The second paragraph exists so the typesetter has more than one block to chew on. Long enough to overflow a single line at the default measure, short enough to keep this fixture quick.

A third paragraph rounds the document out. The point is not the prose; the point is that a corpus-wide grep across the build output finds `pt-line` spans where there used to be plain `<p>` text.
